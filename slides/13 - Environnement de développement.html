Javascript

Environnement de développement

Introduction

Autrefois apprécié pour sa simplicité d'utilisation (un bloc-notes est suffisant), javascript possède désormais un large éventail d'outils d'aide au développement, devenus indispensables pour réaliser des applications riches.


Dans ce chapitre est présenté un exemple d'environnement de développement portable sous Windows.

Git

Logiciel de gestion de versions

  • télécharger la version portable sur le site officiel
  • lancer l'exécutable, créer et choisir le dossier d'installation de votre choix (un répertoire spécifique n'est pas créé automatiquement)

Node.js

Environnement d'exécution javascript

  • télécharger la version Windows Binary (.zip) sur le site officiel
  • l'extraire dans usr/bin du répertoire d'installation de git
  • ouvrir le fichier /etc/bash.bashrc dans le répertoire git avec un éditeur de texte et ajouter en fin de fichier : 
    export PATH=$PATH:/usr/bin/nom_du_repertoire_node/

Visual Studio Code

  • télécharger le zip sur le site officiel et l'extraire dans le répertoire de votre choix
  • ouvrir VSCode (exécutable code.exe)
  • taper F1 pour ouvrir la console de commandes
  • taper settings json et cliquer sur : 
    Préférences: Afficher les paramètres (en JSON)
  • ajouter : 
    {
  "git.path": "C:\\Users\\MonProfil\\Chemin\\vers\\Git\\bin\\git.exe",
  "files.eol" : "\n",
  "editor.codeActionsOnSave": { "source.fixAll.eslint": true },
  "terminal.integrated.profiles.windows": {
    "GitBash": {
      "path": ["C:\\Users\\MonProfil\\Chemin\\vers\\Git\\bin\\bash.exe"],
      "icon": "terminal-bash"
    }
  },
  "terminal.integrated.defaultProfile.windows": "GitBash"
}

Extension eslint

Outil de qualité de code


Dans VSCode :

  • Menu Affichage / Extensions
  • Taper eslint dans le champ de recherche et installer l'extension correspondante.

Création d'un nouveau projet

Création du répertoire

  • créer un répertoire pour votre projet et l'ouvrir dans VSCode par le menu Fichier / Ouvrir le dossier
  • dans VSCode, ouvrir un terminal par le menu Terminal ou Ctrl+ù

Création d'un nouveau projet

Initialisation de Git

  • Dans le terminal taper : 
    git init
  • créer un fichier .gitignore à la racine du projet et ajouter ces lignes : 
    node_modules/
build/

Création d'un nouveau projet

Initialisation NPM et ESLint

  • Dans le terminal taper la commande suivante et répondre aux questions : 
    npm init
  • taper ensuite la commande suivante : 
    npm init @eslint/config

Création d'un nouveau projet

Parcel.js

  • Taper la commande suivante pour installer Parcel.js 
    npm install --save-dev parcel

Création d'un nouveau projet

Ecriture des premiers fichiers

  • créer un répertoire src à la racine du projet
  • créer à l'intérieur le fichier index.html suivant : 
    <!doctype html>
<html lang="fr">
<head>
  <meta charset="utf-8"/>
    <title>Mon projet JS</title>
    <link rel="stylesheet" href="styles.css" />
    <script type="module" src="app.js"></script>
  </head>
  <body>
    <h1>Bonjour le monde</h1>
  </body>
</html>

Création d'un nouveau projet

Ecriture des premiers fichiers

  • créer le fichier styles.css suivant : 
    h1 {
  color: hotpink;
}
  • créer le fichier app.js suivant : 
    console.log('Ça fonctionne');

Création d'un nouveau projet

Lancement de l'application

  • Dans le fichier package.json ajouter dans la section scripts 
    "start": "parcel src/index.html"
  • Dans le terminal, taper : 
    npm start
  • Ouvrir le navigateur à l'adresse http://localhost:1234

Inclusion d'une bibliothèque

Exemple avec lodash

  • Dans le terminal taper la commande suivante 
    npm install lodash
  • Dans le fichier app.js taper : 
    import isEqual from "lodash/isEqual"

console.log(isEqual)

Inclusion d'un fichier JS

  • Créer un fichier src/module1.js 
    export const MY_CONST = "toto"
  • Dans le fichier app.js taper : 
    import { MY_CONST } from "./module1"

console.log(MY_CONST)
